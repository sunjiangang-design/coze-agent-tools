# 伟良日报处理

新乡伟良振动筛工厂每日异常报表自动生成技能。从主计划表和外购物料到货跟踪表中提取异常信息，分类生成结构化报表。

## 输入文件

| # | 文件 | 说明 | 必填 |
|---|------|------|------|
| 1 | 主计划表 | Excel，Sheet名为"订单管理表X-X"格式，数据从第5行开始 | ✅ |
| 2 | 外购物料到货跟踪表 | Excel，Sheet名为"SheetJS" | ✅ |
| 3 | 成品入库表 | Excel，Sheet名为"SheetJS"，当日入库订单涂绿色 | ❌ |

## 输出产物

| # | 产物 | 说明 |
|---|------|------|
| 1 | 异常报表（3个Sheet） | 生产异常 / 采购异常 / 忽略项 |
| 2 | 安全库存未到货表 | 外购物料表中无订单号且欠交>0的行 |
| 3 | 主计划表-成品入库标记 | 可选：当日入库的订单B列涂绿色RGB(169,208,142) |

## 执行流程

### Step 1: 确认参数
- 确认日期（如6.18）
- 确认输入文件路径（主计划表、外购物料表、可选成品入库表）
- 确认主计划表Sheet名（通常为"订单管理表X-X"格式，日期变化）

### Step 2: 运行处理脚本
```bash
cd /app/data/所有对话/主对话
python3 skills/weiliang-daily-report/weiliang_report.py \
  --date 2026-06-18 \
  --main-file "用户上传/飞书/新乡伟良主计划表2026.6.18xxx.xlsx" \
  --purchase-file "用户上传/飞书/外购物料到货跟踪表6.18xxx.xlsx" \
  --output-dir "伟良日报6.18"
```

可选参数：
```bash
  --finished-goods-file "用户上传/飞书/成品入库表6.18xxx.xlsx"
```

### Step 3: 验证输出
- 检查异常报表3个Sheet数据完整性
- 检查安全库存文件字节数>0（曾出现0字节问题）
- 如有成品入库表，检查涂绿色单元格数量

### Step 4: 交付
将输出文件通过 computer:// 协议发送给用户。

## 核心分类规则

### 生产异常 vs 采购异常
- **生产异常**：AV列含"下料、焊接、组装、粘网、机加工"（排除技术/工艺/业务/销售）
- **采购异常**：AV列含"采购"（排除技术/工艺）
- **混合（生产+采购）**：归入采购异常
- **忽略项**：欠计划下单、欠下料排版、欠技术排版、欠排版

### 分类互斥
- 生产异常表去掉采购欠料（筛网/电机/轴承等采购关键字）
- 采购异常表去掉生产异常（焊接/组装/下料等生产关键字）

### 采购关键字
筛网、铸件、圆钢、方管、圆管、卡箍、轴承、电机、螺母、螺栓、尼龙网、网笼丝杆、板带、皮条、软连接、阀门、气缸、气管、弹簧、超声波、电控箱、万向节、同步带、二通、卡簧、三通、平键、堵头、压紧把手、装网工装、手孔盖、感应开关、筛体法兰一

**特殊规则**：物料名含"网笼丝杆"时，"尼龙网"不匹配（优先匹配更长关键字）

### 磅差规则
板材和管材未交货比例 < 7% 时不统计异常（欠交货数量/需求数量 < 0.07 → 忽略）

## 主计划表结构

- 工作Sheet：`订单管理表X-X`（日期变化，需动态获取）
- 数据从第5行开始，第3-4行为列头
- 关键列：B=订单号(2), C=类别(3), D=业务员(4), E=到货地(5), F=机型(6), G=下单日期(7), H=需求日期(8), I=订单数量(9)
- 工序计划列：P=采购(16), R=下料(18), AD=机加工(30), AH=焊接(34), AJ=粘网(36), AL=喷砂(38), AN=喷粉(40), AR=组装/检验(44)
- 实际列 = 计划列 + 1（如S=下料实际19, AI=焊接实际35）
- AU=过程进度异常(47), AV=异常班组(48)

## 外购物料表结构

- 工作Sheet：`SheetJS`
- 关键列：B=采购单号(2), C=任务单号(3), D=物料名称(4), H=采购数量(8), I=已交货(9), J=欠交货(10), K=要求交期(11), L=下单日期(12), M=创建人(13)
- 订单号正则：`^([ZL]\d{4}-\d{1,3}-[YN])`
- 不匹配正则的行 → 安全库存

## 成品入库表结构

- 工作Sheet：`SheetJS`
- 关键列：C=订单编号(3), E=入库时间(5), H=物料名称(8), K=入库数量(11), L=入库状态(12)
- 匹配主计划表B列订单号，当天有入库记录的涂绿色RGB(169,208,142)
- 涂色时只设背景色，不改变字体大小

## AU清洗规则

1. "待" → "欠"
2. 删除冻结报告段（如"5/11报7天冻结"）
3. 按顿号/逗号/换行/分号/斜杠拆分（保护日期中的/）
4. 跳过纯数字/纯日期碎片

### 异常跟踪格式
- 采购欠料：**"欠筛网、电机、弹簧"**（仅第一个加"欠"，其余顿号分隔）
- 自制件：**"欠焊接、组装"**
- 工序未完成：X中→欠X, 已转X→欠X

## 报表格式规范

| 元素 | 字体 | 字号 | 其他 |
|------|------|------|------|
| 主标题 | 宋体 | 18号 | 加粗居中 |
| 行标(第2行) | 宋体 | 12号 | 加粗，填充色RGB(183,222,232)，行高28 |
| 正文 | 宋体 | 12号 | 行高23 |

### 延期天数填色
- 逾期(延期>0)：红色背景白字
- 未来3天未完成(-3~-1)：黄色背景
- 其余：不填色

### 对齐
- 每列居中，"异常跟踪"列左对齐，备注列加宽(≥35)

### 输出前处理
- 取消所有隐藏行（`ws.row_dimensions[r].hidden = False`）
- Sheet命名用点号不用斜杠（如"6.18"）

## 蓝色字体规范

主计划表AU/AV列新追加内容用蓝色字体，需用RichText保留原格式：

```python
from openpyxl.cell.text import InlineFont, RichText, Text
blue_inline = InlineFont(sz=18, color="0000FF", name="宋体")
```

⚠️ InlineFont参数是`sz`不是`size`；Font的`.color.rgb`是8位aRGB，InlineFont的`color`是6位RGB。

## 日期格式规范

日期统一用M/D格式（如6/18）。openpyxl写入datetime后再用`cell.value = string`赋值M/D格式不会生效，必须：
1. 遍历单元格检测`isinstance(val, datetime)`
2. 解析为M/D字符串
3. 同时设`cell.number_format = '@'`强制文本格式

## 注意事项

1. 主计划表Sheet名随日期变化，需动态获取（遍历sheet_names找"订单管理表"开头的）
2. 安全库存文件曾出现0字节问题，生成后务必检查
3. 订单号尾部字母处理：Z2605-17-Yb → 取前缀 Z2605-17-Y
4. 导出xlsx时如遇代理问题，设置`LARK_CLI_NO_PROXY=1`
